# 06-01-PLAN: WebSocket Integration & Session Management

## Description
This plan implements the WebSocket connection and message handling to synchronize game state across multiple devices. It includes a basic Node.js server to relay events.

## Context
Phase 6: Real-time Multiplayer Sync. The app needs to support "Each device only update their score" by broadcasting changes to a shared room.

## Tasks
- [ ] **Task 1: Simple Backend Relay** — Create a minimal Socket.io server (e.g., in a `server/` directory) that handles `join-room` and `transfer-update` events.
- [ ] **Task 2: Socket Client Setup** — Integrate `socket.io-client` into the React application.
- [ ] **Task 3: Room Management UI** — Update `SetupScreen` to allow entering a Room ID or generating a new one.
- [ ] **Task 4: State Synchronization** — Update `performTransfer` in `App.tsx` to emit an event via WebSockets and listen for incoming updates from other devices.
- [ ] **Task 5: Conflict Resolution & Initial Sync** — When a new device joins, it must fetch the current state from other devices in the room.

## Verification
- [ ] Two devices in the same Room ID can see each other's score updates in real-time.
- [ ] Transactions appear in the log on all devices simultaneously.
- [ ] Game history is persisted correctly on all devices when the game ends.
